================================================================================
          PROJECT EXPLANATION: AUTOMATED NOTION-TO-PDF STUDY GUIDE SERVICE
================================================================================

1. THE PROBLEM
--------------------------------------------------------------------------------
Educators and content creators write high-quality study guides and educational 
materials inside Notion. While Notion is an excellent tool for writing, its native 
PDF export options are very limited. 

Directly exporting a Notion page yields a plain, single-column document with:
- Generic fonts and styling.
- Cluttered Table of Contents (TOC) and empty header spaces.
- No corporate branding (like official logo headers or repeating watermarks).
- Basic page numbering and footer layouts.
- Squished tables and plain callout boxes.

To solve this, we needed an automated system that converts raw, exported Notion 
HTML study guides into premium, book-like, print-ready A4 PDFs matching the 
brand design guidelines of Anujjindal.in (brand colors, logo header, watermark, 
two-column layout, custom callouts, and badge page numbers).


2. TYPES OF SOLUTIONS CONSIDERED & COMPARED
--------------------------------------------------------------------------------
A. Manual Graphic Design:
   - How it works: A designer manually copies text into Adobe InDesign or Canva.
   - Pros: High quality.
   - Cons: Slow, expensive, and impossible to scale when processing dozens 
     of pages of study guides daily.

B. Generic PDF Converter Tools:
   - How it works: Online APIs or tools that convert HTML to PDF directly.
   - Pros: Simple.
   - Cons: No control over complex print-media styling, cannot handle custom 
     A4 page breaks, cannot inject custom running headers/footers dynamically, 
     and breaks Notion-specific structures.

C. Headless Browser Compilation (Our Solution):
   - How it works: We use an automated web browser engine (Puppeteer/Chromium) 
     driven by a custom template stylesheet and a JavaScript controller script.
   - Pros: Full control over print margins, exact CSS column styling, dynamic 
     HTML headers/footers, automatic page numbering, and scales instantly.


3. THE TECHNICAL IMPLEMENTATION (HOW WE SOLVED IT)
--------------------------------------------------------------------------------
We built a local toolchain consisting of three core parts:

Part A: The Design System (template_pdf.html)
- Uses CSS Columns (column-count: 2; column-gap: 28px) to format content like 
  a physical textbook.
- Incorporates Google's "Inter" font family for high-quality reading typography.
- Uses HSL brand colors: Primary Blue (#1B71AC) and Primary Green (#2AB573).
- Embeds a subtle background watermark logo centered on every page at 12% opacity.
- Automatically handles Notion tables, making them breakout and span across both 
  columns (column-span: all) so that text is never squished.
- Converts Notion callout boxes into custom-branded "Knowledge Nuggets".

Part B: The Compiler Script (renderer_pdf.js)
- Reads the raw Notion HTML file as a text string.
- Automatically downloads the brand logo and converts it into a "Base64" text 
  string so it renders offline-safely during print.
- Launches an invisible "headless" Google Chrome browser tab via Puppeteer.
- Injects the Notion text into Chrome's memory, cleans up the Table of Contents 
  and header tags, and automatically fixes relative image paths (e.g. converting 
  "Untitled 1.png" into absolute file:// paths so Chrome can load them).
- Calls Chrome's print engine (page.pdf) to generate the A4 output. It injects:
  - Header: Left logo, right subject name.
  - Footer: Left contact phone, center website, right green page number badge.

Part C: The Automation Engine (workflow_pdf.json in n8n)
- A self-hosted n8n workflow that starts a local webhook trigger listener.
- When triggered, it:
  1. Parses the input HTML and output PDF filepaths from the webhook payload.
  2. Runs a shell command to execute "node renderer_pdf.js".
  3. Reads the generated PDF file from disk (supported by setting 
     N8N_RESTRICT_FILE_ACCESS_TO to allow local filesystem access).
  4. Returns the completed PDF binary file directly in the webhook HTTP response.


4. THE FINAL OUTPUT & QUALITY
--------------------------------------------------------------------------------
The generated output is a high-resolution, print-ready A4 PDF document that looks 
like a published textbook:
- The text flows naturally from the left column to the right column, and breaks 
  cleanly across pages without orphan headings (using "break-inside: avoid" rules).
- The company branding is stamped on every page via the header logo, the footer 
  page badge, and the central background watermark.
- The workflow compiles and returns the finished PDF in under 5 seconds!


5. RUNNING THE SYSTEM
--------------------------------------------------------------------------------
A. Running Locally (Direct CLI Command):
   node renderer_pdf.js "/path/to/Notion_Export.html" "/path/to/output.pdf"

B. Running via n8n (Production Webhook Trigger):
   Send a POST request containing the source HTML path and desired output path:

   curl -X POST "http://localhost:5678/webhook/1/webhook/render-pdf" \
     -H "Content-Type: application/json" \
     -d '{
       "inputHtmlPath": "/Users/saieshwarrampelli/Downloads/Anuj Jindal Task/Notes Economic Growth and Development 118b820004a246028d53c0d80e25b5f3.html",
       "outputPath": "/Users/saieshwarrampelli/Downloads/LevelUp/AJCeduttech/output/study_guide.pdf"
     }' \
     --output study_guide.pdf
